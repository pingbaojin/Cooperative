# 合作目标建联（Cooperative Link）使用说明

本文档说明 `data_generate/cooperative_link/` 流水线：**建联前**在雷达点云上做动态目标识别，并与双车 NAV 瞬时先验匹配；**建联后**输出融合相对位姿。支持 **ROS1 在线运行** 与 **bag 离线回放** 两种模式。

---

## 1. 功能概览

| 阶段 | 行为 |
|------|------|
| 未建联 | 先验距离/方位门控 + 载机运动补偿 + 点云聚类 → 与先验匹配 |
| 建联中 (`locking`) | 连续多帧高分检测 → 累计命中 |
| 已建联 (`locked`) | 以双 NAV 几何为主，短时无检测可速度外推 |

**坐标系**：载机雷达系（FLU），`x` 前、`y` 左、`z` 上；与 `build_infos` / CenterPoint 训练数据一致。

**先验来源**：每帧 host + target 的 lat/lon/yaw（ROS topic 或 AVP 文件），**不做** target 轨迹滤波预测。

**多目标（先验扇区内）**：扇区内所有动态簇均识别并发布相对位置（`/cooperative_dynamic/targets`）；仅当某条检测轨迹与 NAV 合作目标轨迹对齐（`cooperative_link_partner_match`）后，才进入 FSM 建联与 `relative_pose` / `relative_pose_filtered` 等后续流程。

## 4. ROS1 标准启动流程

### 4.1 推荐：两终端（launch 含仿真时间与 RViz）

**终端 1 — launch（`use_sim_time`、NAV 发布、建联节点、RViz）**

```bash
source /opt/ros/noetic/setup.bash
cd /docker_ws/ubuntu20_04/Cooperative
source devel/setup.bash
roslaunch cooperative_link play_bag_with_nav.launch
# 无图形界面: roslaunch cooperative_link play_bag_with_nav.launch rviz:=false
# 实车/在线:   roslaunch cooperative_link cooperative_link.launch use_sim_time:=false
```

Launch 参数：`config`、`use_sim_time`（默认 `true`）、`rviz`（默认 `true`）、`rviz_config`。

**终端 2 — 播 bag（必须 `--clock`）**

```bash
source /opt/ros/noetic/setup.bash
rosbag play /media/pbj/Data/UGV1-2025-04-18/2025-04-18-13-59-27.bag --clock
# 与离线一致跳过开头：可加 -s 120
```

### 4.1.1 备选：三终端分步（bag 无 NAV topic）

**终端 1 — NAV 发布**

```bash
source /opt/ros/noetic/setup.bash
cd /docker_ws/ubuntu20_04/Cooperative && source devel/setup.bash
rosrun cooperative_link nav_file_to_ros_publisher _config:=$(rospack find cooperative_link)/config/default.yaml
```

**终端 2 — 播 bag（须先 `rosparam set use_sim_time true` 或使用上方 launch）**

```bash
rosparam set use_sim_time true
rosbag play YOUR.bag --clock
```

**终端 3 — 建联节点**

```bash
rosrun cooperative_link cooperative_link_node _config:=$(rospack find cooperative_link)/config/default.yaml
```

### 4.2 验证

```bash
rostopic hz /clock
rostopic hz /host/navsat          # 约 50Hz
rostopic hz /lslidar_point_cloud
rostopic echo /cooperative_link/state
rostopic echo /cooperative_target/relative_pose_nav
```

### 4.3 RViz（仅球体，无箭头）

默认由 launch 启动（多伙伴用 `cooperative_link_multi.rviz`）；手动：

```bash
# 多伙伴（default.yaml 中 cooperative_link_multi.enabled: true）
rviz -d $(rospack find cooperative_link)/rviz/cooperative_link_multi.rviz

# 单伙伴
rviz -d $(rospack find cooperative_link)/rviz/cooperative_link.rviz
```

| 模式 | MarkerArray 话题 |
|------|------------------|
| 多伙伴 | `/cooperative_target/<link_target_id>/markers`（默认 id=1） |
| 单伙伴 | `/cooperative_target/markers` |

验证 markers 有数据：

```bash
rostopic hz /cooperative_target/1/markers
```

预置配置**只订阅** MarkerArray（球体 + 连线），**不包含** `rviz/Pose` 显示。marker 命名空间为 `coop_nav` / `coop_det` / `coop_fused`。

在 **Global Options → Fixed Frame** 中设为雷达坐标系（与点云 `header.frame_id` 一致，常见 `laser_link`）。

| RViz 显示 | 含义 |
|-----------|------|
| 点云 | `/lslidar_point_cloud` |
| 绿球 + 绿线到原点 | 双 NAV 参考位置 |
| 品红球 | 雷达检测质心（无检测时不显示） |
| 橙/黄球 | 融合位姿（`locking` / `locked` 时） |

**注意**：不要在 RViz 里 Add → **Pose** 并订阅 `/cooperative_target/relative_pose_*`，否则会显示 XYZ 轴箭头。数值位姿请用 `rostopic echo /cooperative_target/relative_pose_nav` 等。

## 5. 输出 Topic 说明

### 5.1 建联节点发布

**全局（单/多伙伴均有）**

| Topic | 类型 | 含义 |
|-------|------|------|
| `/cooperative_targets` | `cooperative_link/CooperativeTargets` | **融合用注册表**：`count`、`partner_ids[]`、`nav_valid[]`、`link_target_id` |

**多目标动态体**（先验门控扇区内全部簇）：

| 话题 | 类型 | 说明 |
|------|------|------|
| `/cooperative_dynamic/targets` | `DynamicTargetArray` | 每帧动态目标相对位姿（host 雷达系，受 `min_motion_score` 过滤） |
| `/cooperative_dynamic/markers` | `MarkerArray` | RViz：**仅稳定 KF 轨迹**（默认最多 6 条）；灰=非队友，金=队友 track；带 `lifetime` 与 DELETE，不会无限堆积 |

`cooperative_link_dynamic` 常用项：`max_dynamic_markers`（RViz 灰球上限）、`min_motion_score`（过滤静态杂簇）、`marker_lifetime_sec`（自动消失时间）、`marker_delete_stale`（对丢失 track 发 DELETE）、`rviz_tracks_only: true`（不为瞬时 DBSCAN 簇画球）。

配置：`cooperative_link_partner_match`（轨迹 DTW 对齐选队友、`require_traj_valid_for_link` 控制是否允许建联输出）。

**单伙伴**（`cooperative_link_multi.enabled: false`）— 扁平话题：

| Topic | 类型 | 含义 |
|-------|------|------|
| `/cooperative_target/relative_pose_nav` | `PoseStamped` | 双 NAV 参考 |
| `/cooperative_target/relative_pose_det` | `PoseStamped` | 雷达估计 |
| `/cooperative_target/relative_pose` | `PoseStamped` | 融合输出（FSM，未卡尔曼滤波） |
| `/cooperative_target/relative_pose_filtered` | `PoseStamped` | 门控簇驱动 CV 卡尔曼 + 多项式（**不用 NAV 作观测**；NAV 仅作 `coord_frame_estimate` 真值） |
| `/cooperative_target/coord_frame_estimate` | `cooperative_link/CoordFrameEstimate` | NAV vs 检测轨迹对齐得到的 2D 旋转/平移误差（**仅诊断**，不写回标定） |
| `/cooperative_target/relative_polar` | `Float32MultiArray` | lidar 系极坐标 |
| `/cooperative_link/state` | `String` | 建联状态 |
| `/cooperative_target/markers` | `MarkerArray` | RViz |
| `/cooperative_uwb/range` | `UwbRange` | 单条 UWB（可选） |

**多伙伴**（`enabled: true`）— 按伙伴 ID 分话题（`{id}` 为 agent ID）：

| Topic | 类型 | 每伙伴 |
|-------|------|--------|
| `/cooperative_target/{id}/relative_pose_nav` | `PoseStamped` | **全部**（仅双 NAV，每帧 NAV 有效即发） |
| `/cooperative_target/{id}/relative_pose_det` | `PoseStamped` | **仅 `link_target_id`**（雷达建联对象） |
| `/cooperative_target/{id}/relative_pose` | `PoseStamped` | 仅 `link_target_id`（FSM 融合） |
| `/cooperative_target/{id}/relative_pose_filtered` | `PoseStamped` | 仅 `link_target_id`（检测观测 KF/多项式，对齐 MATLAB） |
| `/cooperative_target/{id}/coord_frame_estimate` | `CoordFrameEstimate` | 仅 `link_target_id`；窗长满足后 `valid=true` |
| `/cooperative_target/{id}/relative_polar` | `Float32MultiArray` | 仅 `link_target_id` |
| `/cooperative_target/{id}/link_state` | `String` | 仅 `link_target_id` |
| `/cooperative_target/{id}/markers` | `MarkerArray` | 仅 `link_target_id` |
| `/cooperative_uwb/{id}/range` | `UwbRange` | 全部 |
| `/cooperative_uwb/ranges` | `UwbRangeArray` | 聚合（全部伙伴） |

多伙伴下**不再**发布扁平 `/cooperative_target/relative_pose_*`；`publish_legacy_single: false` 时不发 `/cooperative_uwb/range`。

**RViz**：launch 默认加载 `cooperative_link_multi.rviz`（`/cooperative_target/1/markers`）。单伙伴请 `rviz_config:=.../cooperative_link.rviz`。若 `link_target_id` 不是 1，在 RViz 中将 MarkerArray Topic 改为 `/cooperative_target/<id>/markers`。

`CooperativeTargets` 字段：`host_id` 本机 ID；`count` 配置的合作目标总数；`partner_ids` 升序伙伴 agent ID；`nav_valid` 与 `partner_ids` 对齐的本帧 NAV 是否可用；`link_target_id` 当前雷达建联 FSM 使用的伙伴（多伙伴时为配置中第一个）。

```bash
rostopic echo /cooperative_targets
rosmsg show cooperative_link/CooperativeTargets
```

**位姿话题区别**（仅 `link_target_id` 有雷达/KF 链）：

| 话题 | 来源 |
|------|------|
| `relative_pose_nav` | 双 NAV 在 host 雷达系下的参考位置 |
| `relative_pose_det` | 当前帧最佳簇质心 |
| `relative_pose` | FSM 融合（NAV + 检测），建联主输出 |
| `relative_pose_filtered` | 门控簇质心作观测的 CV 卡尔曼 + 可选多项式（与 `Dynamic_Object_Recognition_and_Tracking.m` 一致，**不**把 NAV 送入滤波） |
| `coord_frame_estimate` | 滑动窗口内 **NAV 作真值** vs 滤波轨迹 DTW+SVD 对齐（仅评估，不参与 KF 更新） |

对齐参考系：`reference_frame` 一般为 host 雷达系。参数见 `config/default.yaml` 中 `cooperative_link_filter`、`cooperative_link_calibration`（默认启用；**不会**自动修改 `lidar_yaw_bias` / `R_lidar_vehicle`）。

**短暂丢失（速度递推）**：`cooperative_link_filter.predict_on_miss: true` 时，每帧先 CV 预测（位置 += 速度×dt），无检测的轨在 `max_missed` 帧内仍以 `coasted` 状态输出预测位姿（RViz 略暗、话题 `/cooperative_dynamic/targets` 仍带 `track_id`），队友轨迹匹配缓冲不断档。可调 `max_speed_mps`、`coast_gate_extra_m`（重关联门控放宽）、`max_missed`（默认 8）。

```bash
rostopic hz /cooperative_target/1/relative_pose_filtered
rostopic echo /cooperative_target/1/coord_frame_estimate
```

离线回放 `replay_dynamic_assoc.py` 与线上一致调用后处理；JSONL 含 `filtered_x/y`、`yaw_error_rad`、`tx`、`ty`、`align_cost`。Open3D 可视化中品红球为滤波位姿。

### 5.2 NAV 输入（订阅或文件发布）

| Topic | 类型 | 含义 |
|-------|------|------|
| `/host/navsat` | `sensor_msgs/NavSatFix` | 载机 lat/lon |
| `/host/yaw_deg` | `std_msgs/Float64` | 载机航向（度，北 0、北偏东为正） |
| `/target/navsat` | `NavSatFix` | 目标 lat/lon（单伙伴 legacy） |
| `/target/yaw_deg` | `Float64` | 目标航向（单伙伴 legacy） |
| `/cooperative_partner/{id}/navsat` | `NavSatFix` | 多伙伴模式：伙伴 lat/lon |
| `/cooperative_partner/{id}/yaw_deg` | `Float64` | 多伙伴模式：伙伴航向 |

### 5.3 雷达输入

| Topic | 默认 |
|-------|------|
| `/lslidar_point_cloud` | `sensor_msgs/PointCloud2` |

Topic 名均在 `example_config.yaml` → `cooperative_link_ros` 中可改。

### 5.4 UWB 测距（`cooperative_link_uwb` / `cooperative_link_multi`）

**拓扑**：Host 星形 — 每帧对配置中每个伙伴发布 `from_id=host → to_id=partner` 测距。

| Topic | 类型 | 含义 |
|-------|------|------|
| `/cooperative_uwb/ranges` | `cooperative_link/UwbRangeArray` | **主入口**：本帧所有 host→partner 测距 |
| `/cooperative_uwb/{id}/range` | `cooperative_link/UwbRange` | 可选，按伙伴 ID 分 topic |
| `/cooperative_uwb/range` | `cooperative_link/UwbRange` | 仅单伙伴默认可选（`publish_legacy_single`） |

单条 `UwbRange` 字段：

| 字段 | 含义 |
|------|------|
| `from_id` / `to_id` | 测距端 agent ID（默认 host=0, partner=1） |
| `range_m` | ENU 平面车心距离（米），由双 NAV 仿真 |
| `range_std` | 配置噪声标准差（`range_noise_std`） |
| `valid` | NAV 有效时为 `true` |
| `source` | 固定 `nav_sim` |

**与 `relative_polar` 区别**：

- `relative_polar[0]`：host **lidar 系**下融合位姿的 \(\mathrm{hypot}(x,y)\)
- UWB topic：两车 **ENU 车心**直线距离，供 UWB 因子 / 协同定位使用

**UWB 仅依赖双 NAV 几何**，不要求该伙伴 lidar 建联成功；建联 FSM 仍为单目标（第一个 partner）。合作目标数量与 ID 见 `/cooperative_targets`（与点云同 stamp）。

查看：

```bash
rostopic echo /cooperative_uwb/ranges
rostopic echo /cooperative_uwb/1/range
rosmsg show cooperative_link/UwbRangeArray
rosmsg show cooperative_link/UwbRange
```

配置段（`config/default.yaml`）：

```yaml
cooperative_link_uwb:
  enabled: true
  host_agent_id: 0
  pub_uwb_ranges: /cooperative_uwb/ranges
  pub_uwb_range_per_partner: /cooperative_uwb/{id}/range
  publish_per_partner_topics: true
  publish_legacy_single: true
  range_noise_std: 0.0

cooperative_link_multi:
  enabled: true
  host_agent_id: 0
  partners:
    - id: 1
      name: ugv2
      nav_path: /path/to/ugv2avp.txt
      navsat_topic: /cooperative_partner/1/navsat
      yaw_topic: /cooperative_partner/1/yaw_deg
    - id: 2
      name: ugv3
      nav_path: /path/to/ugv3avp.txt
      navsat_topic: /cooperative_partner/2/navsat
      yaw_topic: /cooperative_partner/2/yaw_deg
```

`cooperative_link_multi.enabled: false` 或未配置 `partners` 时，回退现有单 target（`target_nav_path` + `/target/navsat`），行为与单伙伴一致。

## 6. 配置文件要点

编辑 [`example_config.yaml`](../example_config.yaml)：

```yaml
# 与离线数据生成共用
host_nav_path: ...
target_nav_path: ...
time_offset_sec: 0.0    # nav_time = 雷达 stamp + offset，需与 AVP 第 2 列对齐
bag_path: ...
lidar_topic: /lslidar_point_cloud
R_lidar_vehicle: ...
t_lidar_in_vehicle: ...
gt_center_z_m: 0.0

cooperative_link_ros:
  lidar_topic: /lslidar_point_cloud
  host_navsat_topic: /host/navsat
  # ... 发布 topic 见上文

cooperative_link:
  range_tol_m: 3.0          # 先验距离环带半宽 (m)
  bearing_tol_deg: 15.0     # 方位扇区半角 (deg)
  motion_thresh_m: 0.15     # 运动残差阈值
  min_cluster_points: 8
  lock_frames: 5              # 连续命中帧数 → locked
  score_thresh: 0.6
```

## 9. 常见问题

### Q1: `rostopic echo /host/navsat` 无数据，`Is /clock being published?`

- 使用 launch（默认 `use_sim_time=true`）或执行 `rosparam set use_sim_time true`
- **同时**运行 `rosbag play ... --clock` 与 NAV 发布（launch 或 `nav_file_to_ros_publisher`）
- 仅起发布器、不播 bag 时仿真时间为 0，NAV **不会发布**（AVP 时间为 Unix 秒）

### Q2: `NAV buffer missing data at t_nav=...` / `deque mutated during iteration`

- **`deque mutated during iteration`**：旧版 NAV 环形缓冲无锁，lidar 回调与 NAV 订阅并发会崩溃；请使用含线程锁与快照查询的版本（`nav_ros_buffer._RoleBuffer`）。
- **missing host**：WARN 会带 `reason=`（`empty`/`out_of_range`/`desync`/`clamped`）与缓冲时间跨度；检查四路 NAV、`time_offset_sec`、AVP 是否覆盖当前 bag 时刻。
- navsat 与 yaw **任意到达顺序**均可配对后原子入缓冲（`push_pose`）；略超前于最新 NAV 样本的 lidar 时间在 `cooperative_link_ros.max_query_slop_sec`（默认 **0.15s**）内会钳位插值；仍见 `out_of_range` 时可适当增大该值。
- 联调须 **`use_sim_time=true`** 且 **`rosbag play ... --clock`**；仅起 launch 不播 bag 时 `/clock` 为 0，NAV 不会发布。
- 可用 `rostopic hz /host/navsat`、`/host/yaw_deg` 确认 NAV 在播 bag 时稳定输出（`nav_file_publisher` 默认 50Hz）。

### Q3: `relative_pose_det` 没有输出

- 门控内点过少或未形成簇 → 调大 `range_tol_m` / `bearing_tol_deg` 或减小 `min_cluster_points`
- 查看 `/cooperative_link/state` 是否为 `unlocked`

### Q4: RViz 里出现箭头 / 轴向

- 预置 `cooperative_link.rviz` 已去掉 Pose 显示；若自行添加了 Pose Display，请禁用或删除
- 合作目标位置只看 `/cooperative_target/markers` 球体

### Q5: RViz 里 Marker 位置不对

- **Fixed Frame** 必须与点云 `frame_id` 一致
- 或在 YAML 中设置 `cooperative_link_ros.output_frame_id`

### Q6: bag 里已有 `/fix`、`/GPSpose` 等
